Session 对象

会话状态保持Session对象

Session即会话,是指一个用户在一段时间内对某一个站点的一次访问, 是存储在服务器端的数据。其类名: Httpsession state类

Session

当用户请求一个 ASP. NET页面时,系统将自动创建一个 Session(会话), 退出应用程序或关闭服务器时该会话撒销。针对每一个连接,系统自动分 配一个 SessionID来标识每一个不同用户,该 SessionID在客户端和服务器间 传递,达到唯一标识用户的目的

Session对象特点

  1. Session对象包含某一个用户的状态信息,此信息仅面向该连接,不与其他用户共享
  2. 会话超时或过期,服务器即刻清除 Session对象,释放所占资源
  3. 会话期通过 SessionID传递状态信息,客户端仅对SessionID可见,而对状态信息内容不可见

Session对象使用

xxxxxxxxxx
1. Session的赋值:
    Sessionn["key"]="vlaue";
2. Session获取值:
    变量名=(数据类型)Session["key"]
-------方法/属性-------       
SessionID 获取用于标识会话的唯一会话iD
Clear() 清除会话状态中的所有值
Abandon() 取消当前会话，结束Session会话

Session对象的生命周期,Session是存储在服务器的内存当中 止于网页浏览者关闭此浏览窗口或切断与服务器端的连接。 结東 Session对象生命周期的另一种状况是当浏览者经过一段时间, 并没有持续与服务器端联系,此时会导致变量生命周期结束

2. SqlConnection对象

3. SqlCommand对象

  • 作用: 向数据库发送SQL语句

    • 封装“连接对象"和要执行的"SQL语句"
    • 对数据库执行具体的操作,提供"增、删、改、查"的方法

  • 属性

    • CommandText: 需要封装的sql语句或存储过程名称
    • Connection: Command对象使用的数据库连接对象
    • Parameters: 设置SQL语句或存储过程的参数，可以使用其 Add(stringparameterName) 方法将一个 Parameter 添加到参数集合中

  • 方法

    • ExecuteNonQuery(): 执行增、删、改操作

    • ExecuteScalar(): 返回单一结果的查询, 获取数据库的查询结果(数据类型不确定)

      • 返回值: object类型类型

      • 下拉查看

    • ExecuteReader(): 返回SqlDataReader对象，对象中仅存放一个结果行，可通过读取下一条记录获得新行。

  • ExecuteNonQuery() 方法使用要点

    • 执行insert、update、delete类型的语句
    • 执行后返回受影响的行数，一般是大于0的整数，等于0说明没有影响，-1表示执行有错误

  • 实例化对象及使用

SqlDataAdapter对象

  • 在数据库和DataSet之间执行数据传输的工作。

    • Fill(DataSet.Table)：填充或刷新DataSet中已存在的Table

DataSet对象

  • 方法:

    • Clear():清除所有表中的数据，但保留表结构 。
